    Comment evaluer mon modéle

    Évaluation d'un modèle de régression

    Les coefficients de régression

    L'équation d'une régression linéaire simple peut être écrite comme : 

    y = β0 + β1x + ε
    où :

    • y est la variable dépendante,
    • x est la variable indépendante,
    • β0 est l'interception ou constante,
    • β1 est la pente ou coefficient de régression,
    • ε est l'erreur résiduelle.

    L'équation indique que la valeur de y dépend de la valeur de x multipliée par le coefficient de régression β1 et ajoutée à l'interception β0, avec une certaine erreur résiduelle ε. La régression linéaire multiple peut être étendue pour inclure plusieurs variables indépendantes, ce qui donnerait une équation de la forme :

    y = β0 + β1x1 + β2x2 + ... + βpxp + ε
    où p est le nombre de variables indépendantes.


    Les coefficients de régression sont utilisés pour mesurer la relation entre les variables indépendantes et la variable dépendante dans un modèle de régression. Un coefficient positif indique une relation positive entre les variables, tandis qu'un coefficient négatif indique une relation inverse. En régression linéaire simple, le coefficient de pente mesure la variation de la variable dépendante pour une unité de variation de la variable indépendante, tandis que le coefficient d'interception indique la valeur de la variable dépendante lorsque la variable indépendante est égale à zéro. Cependant, l'interprétation des coefficients dépend du type de modèle et de la nature des données.

    Mean Absolute Error (MAE)

    La MAE mesure l'erreur absolue moyenne entre les prévisions et les observations réelles. Elle est calculée en faisant la somme des valeurs absolues des différences entre chaque prédiction et la valeur réelle, puis en divisant cette somme par le nombre total de prédictions. Elle est souvent utilisée pour des modèles de régression simples, Plus le MAE est faible, meilleure est la performance du modèle.


    L'équation de la MAE est la suivante :

    MAE = (1/n) * Σ|i=1 à n|(|y_i - ŷ_i|)
    Où :

    • n : le nombre d'observations dans l'ensemble de données de test
    • y_i : la valeur réelle de la variable cible pour la i-ème observation
    • ŷ_i : la valeur prédite de la variable cible pour la i-ème observation

    Exemple de code Python pour calculer la MAE :

    from sklearn.metrics import mean_absolute_error

y_true = [3, -0.5, 2, 7]
y_pred = [2.5, 0.0, 2, 8]

mae = mean_absolute_error(y_true, y_pred)

print("MAE: ", mae) #Résultat : MAE: 0.5

    Mean Squared Error (MSE)

    La MSE mesure la moyenne des carrés des erreurs entre les prévisions et les observations réelles. Elle est calculée en faisant la somme des carrés des différences entre chaque prédiction et la valeur réelle, puis en divisant cette somme par le nombre total de prédictions. Elle est souvent utilisée pour des modèles de régression plus complexes. Il est plus sensible aux grandes erreurs que le MAE. Plus le MSE est faible, meilleure est la performance du modèle.


    L'équation de la MSE est la suivante :

    MSE = (1/n) * Σ|i=1 à n|(y_i - ŷ_i)^2
    Où :

    • n : le nombre d'observations dans l'ensemble de données de test
    • y_i : la valeur réelle de la variable cible pour la i-ème observation
    • ŷ_i : la valeur prédite de la variable cible pour la i-ème observation

    Exemple de code Python pour calculer la MSE :

    from sklearn.metrics import mean_squared_error

y_true = [3, -0.5, 2, 7]
y_pred = [2.5, 0.0, 2, 8]

mse = mean_squared_error(y_true, y_pred)

print("MSE: ", mse) #Résultat : MSE: 0.375

    Root Mean Squared Error (RMSE)

    Le RMSE est la racine carrée du MSE et représente l'erreur moyenne entre les prédictions du modèle et les valeurs réelles de la variable cible. Il est souvent utilisé pour des modèles de régression plus complexes. Plus le RMSE est faible, meilleure est la performance du modèle.


    L'équation du RMSE est la suivante :

    RMSE = sqrt(MSE)
    Exemple de code Python pour calculer le RMSE :

    from sklearn.metrics import mean_squared_error
import numpy as np

y_true = [3, -0.5, 2, 7]
y_pred = [2.5, 0.0, 2, 8]

mse = mean_squared_error(y_true, y_pred)
rmse = np.sqrt(mse)

print("RMSE: ", rmse) # Résultat : RMSE: 0.6123724356957945

    Coefficient de détermination R²

    Le coefficient de détermination R² mesure la proportion de variance expliquée par le modèle. Il varie de 0 à 1, où 0 signifie que le modèle ne parvient pas à expliquer la variance de la variable cible et 1 signifie qu'il explique toute la variance. Plus le R² est élevé, meilleure est la performance du modèle. Equation : 

    R² = 1 - SSE / SST
    où SSE est la somme des carrés des résidus (c'est-à-dire la somme des carrés des différences entre les valeurs prédites et les valeurs réelles), et SST est la somme des carrés des différences entre les valeurs réelles et la moyenne des valeurs réelles.

    • Somme des carrés des erreurs (SSE) :

    SSE = Σ(yi - ŷi)²
    où yi est la valeur réelle de la variable dépendante, et ŷi est la valeur prédite par le modèle.

    • Somme totale des carrés (SST) :

    SST = Σ(yi - ȳ)²
    où yi est la valeur réelle de la variable dépendante, et ȳ est la moyenne des valeurs réelles.

    from sklearn.metrics import r2_score

y_true = [3, -0.5, 2, 7]
y_pred = [2.5, 0.0, 2, 8]

# Calcul du coefficient de détermination R²
r2 = r2_score(y_train, y_pred)

print("Coefficient de détermination R² :", r2)

    Interprétation des résultats

    • MAE : Un MAE de 0 signifie que le modèle prédit parfaitement les valeurs de la variable cible. En général, un MAE faible indique une bonne performance, mais cela dépend du contexte et de la plage de valeurs de la variable cible.

    • MSE et RMSE : Comme mentionné précédemment, ces deux métriques sont plus sensibles aux grandes erreurs que le MAE. Par conséquent, un MSE ou un RMSE élevé peut indiquer qu'il y a des valeurs aberrantes dans les données ou que le modèle ne parvient pas à prédire correctement les valeurs extrêmes de la variable cible.

    • R² : indique la proportion de la variance totale des données qui est expliquée par le modèle. Un R² de 1 signifie que le modèle explique parfaitement la variance des données, tandis qu'un R² de 0 signifie que le modèle n'explique aucune variance des données. Dans la pratique, un R² élevé (généralement supérieur à 0,7 ou 0,8) est considéré comme un bon ajustement du modèle aux données.

    • SSE indique la quantité d'erreur résiduelle dans le modèle. En général, plus SSE est faible, mieux c'est, car cela signifie que les prédictions du modèle sont plus proches des valeurs réelles.

    • SST indique la quantité de variation totale dans les données. Il est souvent utilisé comme mesure de la variabilité des données de référence. Un modèle qui explique une grande partie de la variation totale de SST est considéré comme un bon ajustement aux données.

    Évaluation d'un modèle de classification

    Accuracy

    L'accuracy est la métrique la plus simple pour évaluer les modèles de classification. Elle mesure la proportion de prédictions correctes sur l'ensemble de données de test.


    L'équation de l'accuracy est la suivante :

    Accuracy = (TP + TN) / (TP + TN + FP + FN)
    Où :

    • TP : True Positives, le nombre de prédictions positives correctes
    • TN : True Negatives, le nombre de prédictions négatives correctes
    • FP : False Positives, le nombre de prédictions positives incorrectes
    • FN : False Negatives, le nombre de prédictions négatives incorrectes

    Exemple de code Python pour calculer l'accuracy :

    from sklearn.metrics import accuracy_score

y_true = [0, 1, 2, 0, 1, 2]
y_pred = [0, 2, 1, 0, 0, 1]

accuracy = accuracy_score(y_true, y_pred)

print("Accuracy: ", accuracy) #Résultat : Accuracy: 0.3333333333333333

    Precision, Recall et F1-score

    La précision (precision), le rappel (recall) et le score F1 (F1-score) sont des métriques plus avancées pour évaluer les modèles de classification. Elles sont particulièrement utiles pour les problèmes de classification binaire, où l'on souhaite prédire la présence ou l'absence d'une classe.

    • La précision mesure la proportion de prédictions positives qui sont correctes. Elle est calculée en divisant le nombre de vrais positifs par le nombre de vrais positifs plus le nombre de faux positifs.L'équation de la précision est la suivante : 
      Precision = TP / (TP + FP)
    • Le rappel mesure la proportion de vrais positifs qui sont correctement identifiés. Il est calculé en divisant le nombre de vrais positifs par le nombre de vrais positifs plus le nombre de faux négatifs. L'équation du rappel est la suivante :

    Recall = TP / (TP + FN)
    - Le score F1 est une moyenne harmonique de la précision et du rappel. Il est calculé en prenant la moyenne pondérée de la précision et du rappel, où la pondération est donnée par l'harmonique de la précision et du rappel. L'équation du score F1 est la suivante :

    F1 = 2 * (Precision * Recall) / (Precision + Recall)
    Exemple de code Python pour calculer la précision, le rappel et le score F1 :

    from sklearn.metrics import precision_score, recall_score, f1_score

y_true = [0, 1, 2, 0, 1, 2]
y_pred = [0, 2, 1, 0, 0, 1]

precision = precision_score(y_true, y_pred, average='macro')
recall = recall_score(y_true, y_pred, average='macro')
f1 = f1_score(y_true, y_pred, average='macro')

print("Precision: ", precision)
print("Recall: ", recall)
print("F1-score: ", f1)

    Matrice de confusion

    La matrice de confusion est un outil visuel pour évaluer les performances d'un modèle de classification. Elle montre le nombre de prédictions correctes et incorrectes pour chaque classe.


    Exemple de code Python pour afficher la matrice de confusion :

    from sklearn.metrics import confusion_matrix
import matplotlib.pyplot as plt
import seaborn as sns

y_true = [0, 1, 2, 0, 1, 2]
y_pred = [0, 2, 1, 0, 0, 1]

cm = confusion_matrix(y_true, y_pred)

sns.heatmap(cm, annot=True, cmap="Blues")
plt.xlabel('Prédictions')
plt.ylabel('Valeurs réelles')
plt.show()

    Interpréter les métriques d'évaluation de classification:

    Tout dépend du contexte spécifique de chaque projet de machine learning. Cependant, voici quelques points généraux à prendre en compte lors de l'interprétation de ces métriques :

    • Précision : La précision mesure le taux de prédictions positives correctes parmi toutes les prédictions positives. Par exemple, une précision de 0,8 signifie que le modèle a correctement prédit 80% des résultats positifs. Cependant, si la précision est élevée mais que le rappel est faible, cela peut indiquer que le modèle prédit trop peu de résultats positifs, manquant ainsi des résultats positifs réels.

    • Rappel : Le rappel mesure le taux de prédictions positives correctes parmi tous les résultats positifs réels. Par exemple, un rappel de 0,7 signifie que le modèle a correctement prédit 70% des résultats positifs. Cependant, si le rappel est élevé mais que la précision est faible, cela peut indiquer que le modèle prédit trop de résultats positifs, y compris des résultats faux positifs.

    • Score F1 : Le score F1 est une moyenne harmonique de la précision et du rappel. Il est généralement utilisé lorsque l'on souhaite trouver un équilibre entre la précision et le rappel.

    • Matrice de confusion : La matrice de confusion montre le nombre de prédictions correctes et incorrectes pour chaque classe. Elle permet de voir où le modèle est en train de se tromper. Par exemple, si le modèle prédit souvent la classe 0 alors qu'elle est réellement la classe 1, il peut y avoir une confusion entre ces deux classes.

    En général, il est important d'évaluer le modèle avec plusieurs métriques pour avoir une vue d'ensemble de ses performances. Selon le contexte, certaines métriques peuvent être plus importantes que d'autres. Par exemple, dans les problèmes de détection de fraude, un rappel élevé peut être plus important que la précision, car il est crucial de ne pas manquer de cas de fraude, même si cela signifie avoir plus de faux positifs.
