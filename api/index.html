    API (Application Programmable Interface)

    API (Application Programming Interface)

    C'est quoi ?

    API (Application Programmable Interface) est un ensemble de fonctions permettant d'utiliser les services d’une application. Une API est peut être distribué localement dans un programme informatique (accessible uniquement par un algorithme), ou au contraire peut avoir vocation a être utilisée par un plus grand nombre d'acteurs.


    Il y a différents type d'API. Dans ce cours, nous nous intéressons surtout au API Web, c'est-à-dire celles qui permettent de fournir une interface accessible en ligne. Cela est le cas lorsque l'on effectue une requête à un serveur afin que l'on reçoive le résultat d'un traitement.

    Les API RESTful

    Une API RESTful (Representational State Transfer) est une architecture d'API Web qui repose sur le protocole HTTP et qui utilise ses méthodes (GET, POST, PUT, DELETE) pour permettre aux clients de communiquer avec les serveurs et de récupérer des données. Les API RESTful sont largement utilisées dans les applications Web et mobiles pour fournir une interface standardisée et facile à utiliser pour accéder aux données.



    Voici les principaux concepts de l'architecture RESTful :


    • Ressources : les ressources sont les entités exposées par l'API, telles que les utilisateurs, les produits, les commandes, etc.

    • Verbes HTTP : les verbes HTTP (GET, POST, PUT, DELETE) sont utilisés pour décrire les opérations sur les ressources. Le verbe GET est utilisé pour récupérer une ressource, POST pour créer une nouvelle ressource, PUT pour mettre à jour une ressource existante et DELETE pour supprimer une ressource.

    • Les routes : Les routes dans une RESTful API sont des points d'entrée spécifiques dans l'API qui définissent les opérations disponibles et les ressources accessibles. Une route est généralement un URI (Uniform Resource Identifier) qui identifie de manière unique une ressource et une méthode HTTP qui spécifie l'action à effectuer sur cette ressource.

    • Représentations : les représentations sont les formats de données utilisés pour représenter les ressources, tels que JSON, XML, ou YAML. Le format le plus couramment utilisé est le JSON (JavaScript Object Notation), car il est léger, facile à lire et à écrire.

    • Les codes de réponse HTTP : les codes de réponse HTTP sont utilisés pour indiquer le résultat d'une opération sur une ressource. Par exemple, le code 200 indique que la demande a été traitée avec succès, le code 201 indique que la ressource a été créée avec succès, le code 404 indique que la ressource n'a pas été trouvée, etc.

    En utilisant ces concepts, une API RESTful peut fournir une interface simple, cohérente et facile à utiliser pour accéder aux données et aux services d'un serveur. Les clients peuvent utiliser des requêtes HTTP standard pour accéder aux ressources et interagir avec le serveur, ce qui rend l'API RESTful très flexible et facile à intégrer dans des applications Web et mobiles.

    Flask

    Flask est une bibliothèque Python qui permet de créer des applications Web et des API RESTful. Voici un exemple de code pour créer une API RESTful simple avec Flask :

    from flask import Flask, jsonify, request

app = Flask(__name__)

# Définition de quelques données pour notre API
students = [
    {
        'id': 1,
        'name': 'Alice',
        'age': 20
    },
    {
        'id': 2,
        'name': 'Bob',
        'age': 22
    },
    {
        'id': 3,
        'name': 'Charlie',
        'age': 21
    }
]

# Route pour récupérer tous les étudiants
@app.route('/students', methods=['GET'])
def get_students():
    return jsonify(students)

# Route pour récupérer un étudiant en particulier
@app.route('/students/<int:id>', methods=['GET'])
def get_student(id):
    student = [student for student in students if student['id'] == id]
    return jsonify(student)

# Route pour créer un nouvel étudiant
@app.route('/students', methods=['POST'])
def create_student():
    student = {
        'id': request.json['id'],
        'name': request.json['name'],
        'age': request.json['age']
    }
    students.append(student)
    return jsonify(student)

# Route pour mettre à jour un étudiant existant
@app.route('/students/<int:id>', methods=['PUT'])
def update_student(id):
    student = [student for student in students if student['id'] == id]
    student[0]['name'] = request.json.get('name', student[0]['name'])
    student[0]['age'] = request.json.get('age', student[0]['age'])
    return jsonify(student[0])

# Route pour supprimer un étudiant existant
@app.route('/students/<int:id>', methods=['DELETE'])
def delete_student(id):
    student = [student for student in students if student['id'] == id]
    students.remove(student[0])
    return jsonify({'result': True})

# Lancement de l'application Flask
if __name__ == '__main__':
    app.run(debug=True)

    Dans cet exemple, nous avons créé une API RESTful simple qui permet de récupérer, créer, mettre à jour et supprimer des étudiants. Nous avons utilisé les décorateurs @app.route pour définir les différentes routes de notre API, ainsi que les méthodes HTTP correspondantes (GET, POST, PUT, DELETE). Nous avons également utilisé la fonction jsonify de Flask pour retourner des données JSON à partir de notre API.


    Pour tester notre API, nous pouvons utiliser un outil comme Postman ou simplement effectuer des requêtes HTTP en utilisant l'URL de notre API (par exemple http://localhost:5000/students pour récupérer tous les étudiants).
